## 🎯 نظام الكريدت

### أنواع المستخدمين

| النوع | الوصف | الخصم |
|-------|-------|-------|
| **Admin** | صلاحيات كاملة | لا يوجد خصم |
| **Unlimited Premium** | وصول غير محدود | لا يوجد خصم |
| **Credit Premium** | اشتراك بالنقاط | 1 نقطة لكل فحص |
| **Free** | مستخدم مجاني | مهلة 30 ثانية في المجموعات |

### كيفية عمل النظام

#### الفحص الفردي
1. المستخدم يرسل أمر فحص (مثل `/b3`, `/vbv`, إلخ)
2. النظام يتحقق عبر `config.can_use_b3()`
3. إذا كان Admin أو Unlimited → السماح مباشرة
4. إذا كان Credit Premium → خصم نقطة واحدة
5. إذا نفد الرصيد → رسالة خطأ
6. عند نجاح الفحص → تحديث `last_chk`

#### الفحص الجماعي
1. المستخدم يرفع ملف بطاقات
2. النظام يتحقق من وجود رصيد عبر `can_start_mass()`
3. لكل بطاقة → خصم نقطة واحدة
4. عند نفاد الرصيد → توقف تلقائي
5. عند خطأ مؤقت → استرجاع النقطة (Refund)

## 🔧 إدارة الكريدت

### إضافة كريدت لمستخدم
1. `/admin`
2. اختر "💰 Add Credits"
3. أدخل الكمية (مثلاً: 50)
4. أدخل user_id
5. ✅ تم الإضافة

### تفعيل Unlimited
1. `/admin`
2. اختر "♾️ Set Unlimited"
3. أدخل user_id
4. ✅ تم التفعيل

### إلغاء Unlimited
1. `/admin`
2. اختر "♾️ Set Unlimited"
3. أدخل: `remove USER_ID`
4. ✅ تم الإلغاء

## ⚠️ ملاحظات مهمة

### التوافق مع النظام القديم
- النظام الجديد يدعم الاشتراكات الزمنية القديمة
- المستخدمون القدامى يستمرون في العمل حتى انتهاء اشتراكهم
- يمكن إضافة كريدت لمستخدم لديه اشتراك زمني

### الأولوية
```
1. Admin (أعلى أولوية)
2. Unlimited
3. Credits
4. Time-based subscription
5. Free (أقل أولوية)
```

### سياسة Refund
| الحالة | الفعل |
|--------|-------|
| Timeout / Network / TLS / 5xx | Refund 1 |
| خطأ إدخال (بطاقة خاطئة) | لا خصم |
| Declined / Charged / Live | الخصم يبقى |
