# محاكاة تفاعلية: سيناريوهات المستخدم العادي

هذا القسم يوضح محاكاة تفاعلية لجميع الأوامر والأزرار التي يمكن للمستخدم العادي (غير المشرف) التفاعل معها في البوت.

## 1. التفاعل الأولي والواجهة (الأوامر: /start و /cmds)

| الخطوة | الإجراء (المستخدم) | الرد (البوت) | الدالة/الزر المرتبط |
| :--- | :--- | :--- | :--- |
| **1. البدء** | إرسال الأمر `/start` | **رسالة ترحيب:** `Good Noon 🌅 User... Press /cmds To Know My Features` مع صورة بانر. | `start_command` |
| **2. الواجهة الرئيسية** | الضغط على زر **👤 MY PROFILE** | **تحديث الرسالة:** عرض معلومات المستخدم والاشتراك (UID، Name، Status، Credits، Validity، Last Chk). | `start_callbacks` (st:profile) |
| **3. العودة** | الضغط على زر **⬅️ BACK** | **تحديث الرسالة:** العودة إلى رسالة الترحيب الرئيسية. | `start_callbacks` (st:home) |
| **4. قائمة الأوامر** | إرسال الأمر `/cmds` | **رسالة لوحة التحكم:** `Commands Overview... Click below to view full details.` مع صورة بانر جديدة. | `cmds_entry` |
| **5. استعراض البوابات** | الضغط على زر **🧭 GATEWAYS** | **تحديث الرسالة:** `Gateways — Choose a category.` مع أزرار الفئات: **✅ AUTH**، **⚡ CHARGE**، **🔒 VBV/3DS**. | `cmds_callbacks` (cmds:gates) |
| **6. استعراض الأدوات** | الضغط على زر **🛠 TOOLS** | **تحديث الرسالة:** `Tools Menu — Legacy commands` مع أزرار **CC Generate** و **CC Scraper**. | `cmds_callbacks` (cmds:tools) |
| **7. تفاصيل التوليد** | الضغط على زر **CC Generate** | **تحديث الرسالة:** عرض صيغة الأمر `/gen` مع مثال: `/gen 42424242 |12|2024|5`. | `cmds_callbacks` (tools:gen) |
| **8. تفاصيل الفحص** | الضغط على زر **CC Scraper** | **تحديث الرسالة:** عرض صيغة الأمر `/scr` مع مثال: `/scr Mustafa2l 50`. | `cmds_callbacks` (tools:scr) |

## 2. محاكاة فحص البطاقات الفردي (Single Check)

**السيناريو:** مستخدم يرسل بطاقة وهمية للفحص عبر بوابة `B3 Gateway` (`/b3`).

| الخطوة | الإجراء (المستخدم) | الرد (البوت) | الملاحظات (نظام الائتمانات) |
| :--- | :--- | :--- | :--- |
| **1. فحص مجاني** | `/b3 4000000000000000|12|2025|123` | **الرد:** `[B3 Gateway] ⏳ Checking Card...` ثم `✅ LIVE! - Approved (Mock Response)` | **المستخدم المجاني:** يتم تطبيق فترة تهدئة (Cooldown) مدتها 30 ثانية قبل السماح بفحص آخر. |
| **2. فحص مميز** | `/b3 5000000000000000|01|2026|456` | **الرد:** `[B3 Gateway] ⏳ Checking Card...` ثم `❌ DEAD! - Declined (Mock Response)` | **المستخدم المميز:** يتم استدعاء `consume_for_single` لخصم ائتمان واحد. |
| **3. خطأ مؤقت** | `/b3 4111111111111111|05|2024|789` | **الرد:** `⚠️ UNK! - Gateway Timeout (Refunded)` | يتم استدعاء `refund_credit` لإعادة الائتمان للمستخدم بسبب الخطأ المؤقت. |
| **4. نفاد الائتمان** | `/b3 ...` (بعد نفاد الائتمان) | **الرد:** `⛔ لا يوجد كريدت كافٍ لإجراء الفحص.` | يتم منع الفحص، ويتم توجيه المستخدم لشحن رصيده. |

## 3. محاكاة الفحص الجماعي (Mass Check)

**السيناريو:** مستخدم مميز لديه رصيد كافٍ يقوم بإجراء فحص جماعي باستخدام الأمر `/mass`.

| الخطوة | الإجراء (المستخدم) | الرد (البوت) | الدالة المرتبطة |
| :--- | :--- | :--- | :--- |
| **1. بدء الفحص** | إرسال الأمر `/mass` | **الرد:** `✅ يرجى إرسال ملف البطاقات (.txt) الآن.` | `handle_mass_command` |
| **2. إرسال الملف** | إرسال ملف `cards.txt` (يحتوي على 10 بطاقات وهمية) | **الرد:** `✅ تم استلام الملف (10 بطاقات وهمية). يرجى اختيار البوابة التي تريد استخدامها:` مع أزرار البوابات. | `handle_document` |
| **3. اختيار البوابة** | الضغط على زر `/b3` | **الرد:** `⏳ جاري بدء الفحص الجماعي...` (رسالة حالة قابلة للتحديث). | `mass_callback_handler` |
| **4. سير الفحص** | (البوت يعمل في الخلفية) | **تحديث رسالة الحالة:** `📊 حالة الفحص الجماعي... البطاقات: 5/10... Live: 2, Dead: 3` | `start_mass_check` و `update_status_message` |
| **5. نتائج فردية** | (البوت يعمل في الخلفية) | **رسائل متتالية:** `✅ Approved... Card: 4000...`، `❌ Declined... Card: 5000...` | `check_single_card_mock` |
| **6. انتهاء الفحص** | (بعد الانتهاء من 10 بطاقات) | **تحديث رسالة الحالة النهائية:** `📊 حالة الفحص الجماعي... البطاقات: 10/10... الحالة: FINISHED`، ثم رسالة `✅ انتهت جلسة الفحص الجماعي.` | `start_mass_check` |
| **7. نفاد الائتمان أثناء الفحص** | (إذا نفد الائتمان عند البطاقة رقم 7) | **رسالة نتيجة:** `⛔ Card: 4666... ⇾ نفاد الكريدت. توقفت الجلسة.` | يتم تحديث حالة الجلسة إلى `stopped_no_credit`، ويتوقف الفحص. |
